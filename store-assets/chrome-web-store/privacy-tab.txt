==================================================
SINGLE PURPOSE  (one text box)
==================================================
Compliance Central lets Michigan automotive dealers screen buyers and co-buyers for OFAC sanctions, MDOS repeat-offender status, and vehicle title/lien status from a single browser side panel, and produce a printable record of the result.


==================================================
PERMISSION JUSTIFICATIONS  (one box per permission)
==================================================

sidePanel —
The entire user interface is a Chrome side panel; this permission opens and renders it.

storage —
Saves preferences and bounded anonymous audit history (up to 30 days / 50 records). Persistent records contain an anonymous reference, timestamp, overall decision, check outcomes, and co-buyer/trade-in workflow flags; they exclude customer identity, VINs, screenshots, and full report payloads.

unlimitedStorage —
Stores the downloaded OFAC SDN sanctions dataset for on-device screening and bounded anonymous audit history. Customer identity, VINs, full results, and portal screenshots remain session-only.

alarms —
Schedules a daily attempt to refresh the local OFAC sanctions list and maintain the 30-day history limit.

Host permission https://sanctionslistservice.ofac.treas.gov/ —
Downloads the official U.S. Treasury OFAC SDN sanctions list used for on-device screening.

Host permission https://wc2h-sls-prod-public-published.s3.us-gov-west-1.amazonaws.com/ —
Allows the official Treasury Sanctions List Service download to follow its signed redirect to Treasury's dedicated AWS GovCloud file host. Only the public OFAC list is downloaded.

Host permission https://compliance-central-api.fly.dev/ —
The dealer's backend endpoint that performs the Repeat Offender and Title/Lien checks against the Michigan Department of State portal. It also relays the optional phone license-scan: the scanned fields are end-to-end encrypted on the phone, so the backend only passes along an encrypted, single-use package it cannot read (deleted on pickup; the license image is never transmitted).

The hosting service receives ordinary network request metadata. Compliance Central uses request IP addresses transiently in memory for rate limiting and abuse prevention; the app does not write them to a database or application log.

Are you using remote code?  →  NO
(All code is bundled in the package; nothing is fetched and executed remotely.)

==================================================
LIMITED USE DISCLOSURE
==================================================
Compliance Central's use of information received from Google APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Information is used only to provide the extension's user-requested compliance checks, phone-to-computer transfer, local preferences, and anonymous audit records. It is not used for advertising, market research, credit scoring, or unrelated purposes. Human access is limited to a user's explicit support request, security investigation, legal requirement, or aggregated operations that cannot identify a person.
